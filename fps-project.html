    Project Goals & Progress

    Working with a level designer friend, we wanted to create an FPS mission that takes inspiration from games like Wolfenstein. The goal is to have dual wielding weapons, the option of stealth, and some set piece moments.

    This project has not been active very long, but we are planning on having the player traverse through a train station area, before boarding the train and fighting on it. So far, I have been building the base systems and player-related features in C++ in Unreal.

    Combat System

    I've used C++ before, but not in Unreal, so this is my first time. I want to understand it better though, so I am building all the base classes starting in C++ first. After getting the first-person character set up, I began on the combat system.

    Weapons

    We really wanted to focus our combat around fun weapons, including the ability to mix and match dual-wielding them. To do this I created a weapon component that contains a switchable WeaponDataAsset. I then attached two weapon components to the player, and hooked up the player input to control each hands weapon.

    When the player equips a weapon, I can switch the WeaponDataAsset with the correct data. This means it is decoupled from the WeaponComponent itself and changing a weapon stats independently is easy.

    View WeaponComponent Script

    Ammo & Inventory

    Since different weapons can have the same ammo type, I made a Combat Component to compliment the two Weapon Components. This became the authority over the player and holds all information related to ammo reserves. So, when the player requests to reload their weapon, it checks the Combat Component to see what is available.

    I did this because it removes the need for each WeaponDataAsset to know the amount of ammo a weapon currently has, and the component will also become more useful in the future with added player abilities or inventory items.

    View CombatComponent Script

    Firing

    This is still a work-in-progress, but we are going for the design of having no bullets actively spawn in. When you fire your weapon, it draws a small sphere trace and adds some random spread (depending on adjustable variables) to check for immediate hits.

    I will have to use effects to make this believable in the future, but for now it detects hits and will damage enemies or activate effects once we have created those.
